# 多跳检索指南（Multi-hop Retrieval）

本项目支持 **多跳检索**：在单跳向量检索基础上，根据第 1 跳结果生成子查询，再进行第 2 跳（及更多跳）检索，合并去重后作为 RAG 上下文。适用于需要跨文档、多步关联的风控问题。

> **默认关闭**（`risk-ai.multi-hop.enabled=false`），与改造前单跳行为完全一致，不影响现有接口。

---

## 1. 什么时候需要多跳？

| 场景 | 单跳 | 多跳 |
|------|------|------|
| 「员工年假有多少天？」 | ✅ 通常足够 | 可选 |
| 「年假规定及审批流程是什么？」 | 可能只命中一半 | ✅ 第 2 跳补「审批流程」 |
| 「反欺诈规则与合规处罚的关系？」 | 易漏关联文档 | ✅ 子查询分别检索 |

---

## 2. 工作流程

```
用户问题
    ↓
第 1 跳：用原始问题做向量检索（hopTopK 条）
    ↓
生成子查询（LLM 或规则模板，最多 subQueriesPerHop 条）
    ↓
第 2 跳：对每个子查询再检索
    ↓
按 chunk id 合并去重，保留较高相似度
    ↓
EnhancedRetrievalService.refine()（混合检索 + Rerank 精排）
    ↓
取 finalTopK 条 → 交给 RagRagentService 拼 Prompt + 调大模型
```

### 子查询生成方式

| 模式 | 配置 | 说明 |
|------|------|------|
| LLM 扩展 | `use-llm-sub-query: true`（默认） | 根据第 1 跳片段，让千问生成补充检索词 |
| 规则模板 | `use-llm-sub-query: false` | 自动追加「相关规定」「审批流程」「实施细则」等后缀 |

LLM 失败时自动降级为规则模板。

## 3. 配置项

`application.yml` → `risk-ai.multi-hop.*`（由 `RagProperties.MultiHop` 绑定）：

| 配置 | 默认 | 说明 |
|------|------|------|
| `enabled` | **false** | 是否启用多跳（改为 `true` 即开启） |
| `max-hops` | 2 | 最大跳数（含第 1 跳原问题） |
| `sub-queries-per-hop` | 2 | 每一跳（第 2 跳起）最多子查询数 |
| `hop-top-k` | 3 | 每一跳向量检索条数 |
| `final-top-k` | 5 | 合并去重后保留条数 |
| `use-llm-sub-query` | true | 是否用大模型生成子查询 |

### 开启示例

```yaml
risk-ai:
  multi-hop:
    enabled: true
    max-hops: 2
    sub-queries-per-hop: 2
    hop-top-k: 3
    final-top-k: 5
    use-llm-sub-query: true
```

修改后重启后端生效。

## 4. 影响范围

| 路径 | 是否走多跳 |
|------|------------|
| `POST /risk/ragent` | ✅（`enabled=true` 时） |
| `POST /api/user/chat` | ✅ |
| `POST /api/admin/ragent/ask` | ✅ |
| MCP `searchRiskKnowledge` | ❌ 不走多跳，但走混合检索 + Rerank |
| MCP `askRiskQuestion` | ✅（走 `RagRagentService`，受配置影响） |

多跳与混合检索、Rerank 可叠加使用，详见 [混合检索与Rerank指南.md](混合检索与Rerank指南.md)。

---

## 5. 响应字段

问答接口 `RagentResponse` 新增（向后兼容，默认 `multiHopUsed=false`、`retrievalHops=1`）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `multiHopUsed` | boolean | 是否使用了多跳检索 |
| `retrievalHops` | int | 实际执行的跳数 |

示例：

```json
{
  "code": 200,
  "data": {
    "answer": "...",
    "multiHopUsed": true,
    "retrievalHops": 2,
    "costMs": 3200
  }
}
```

## 6. 相关代码

| 文件 | 职责 |
|------|------|
| `service/MultiHopRetrievalService.java` | 多跳主逻辑 |
| `service/EnhancedRetrievalService.java` | 多跳合并后的混合检索 + Rerank |
| `service/RagRagentService.java` | 调用 `multiHopRetrievalService.retrieve()` |
| `service/VectorStoreService.java` | 单跳/每跳向量检索（支持自定义 topK） |
| `config/RagProperties.java` | `MultiHop` 配置绑定 |
| `dto/MultiHopRetrievalResult.java` | 内部检索结果 |

---

## 7. 注意事项

1. **多跳会多调 1 次大模型**（生成子查询时），耗时与 Token 消耗高于单跳。
2. 开启多跳后，答案缓存 Key 会附加 `|mh` 后缀，与单跳缓存隔离。
3. 第 1 跳无结果时，不会继续后续跳，直接返回空上下文。
4. 生产环境建议先在小流量验证 `final-top-k` 与 `max-hops`，避免上下文过长。
5. **多跳关闭时**，问答仍默认走混合检索 + Rerank（见 `risk-ai.retrieval.*`）。
