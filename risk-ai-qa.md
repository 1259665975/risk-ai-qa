# 金融风控智能问答系统（RAG）需求文档
## 一、技术栈
- 运行环境：Java 17
- 项目框架：Spring Boot 3.2.x
- AI 组件：Spring AI 1.0.x
- 向量数据库：Milvus 2.x
- 缓存/限流：Redis
- 数据持久化：MySQL 8.0
- 文档解析：Apache Tika
- 大模型：阿里云通义千问 API
- 构建工具：Maven

## 二、整体说明
基于 RAG 检索增强生成架构，打造金融风控私有化知识库问答系统。
面向信贷风控、反欺诈、合规场景，**严格禁止大模型编造内容**，所有回答必须基于上传的私有文档，满足金融合规要求。

## 三、核心功能模块
### 1. 知识库管理模块
1. 支持解析格式：TXT、PDF 文档[README.md](README.md)
2. 文本处理：自动过滤空行、多余空格、特殊符号，完成内容清洗
3. 切片规则：单段 800 Token，上下文重叠 150 Token，保证语义完整
4. 向量处理：调用 Embedding 接口生成向量，存入 Milvus 向量库
5. 基础功能：文档批量入库、知识库全量清空、支持增量更新

### 2. RAG 智能问答核心模块
1. 接口地址：`POST /risk/qa`
2. 请求参数：question（用户提问，非空，长度限制 2~200 字）
3. 执行流程：
   - 校验请求参数
   - 用户问题向量化
   - Milvus 相似度检索，取出 Top3 匹配文档片段
   - 拼接【系统提示词+检索文档+用户问题】组装完整 Prompt
   - 调用通义千问大模型生成回答
   - 结果格式化后返回前端
4. 强制 Prompt 规则：
> 你是专业的金融风控合规顾问，仅依托提供的内部风控参考文档回答用户问题。严格遵守以下规则：
> 1、所有回答必须100%溯源参考文档内容，文档无对应信息时，统一回复“暂无相关风控规则信息”，严禁编造、推演、猜测内容；
> 2、回答语言简洁、专业、严谨，贴合金融风控业务规范，不输出口语化、无关、冗余内容；
> 3、仅解答风控准入、审批规则、反欺诈、合规相关问题，非业务问题统一礼貌拒绝回复；
> 4、严禁输出违规、不合规、超出企业内部制度的内容。

### 3. 高可用防护模块
1. IP 限流（Redis 实现）：同一IP 1分钟内最多访问 10 次问答接口，超限返回 429 状态码+提示
2. 结果缓存（Redis 实现）：高频问答结果缓存 24 小时，命中缓存直接返回，减少模型调用
3. 服务降级：大模型超时、网络异常、API 报错时，统一返回兜底文案：`AI问答服务暂时繁忙，请稍后重试`

### 4. 问答日志模块（MySQL）
1. 数据表字段：日志ID、提问内容、回答内容、客户端IP、请求时间、响应耗时、是否命中缓存、检索文档片段、请求状态
2. 功能：接口调用后异步写入日志，用于运维审计、知识库优化、问题统计

## 四、对外接口清单
1. `POST /doc/ingest` ：上传/读取本地文档，完成解析、切片、向量入库
2. `POST /risk/qa` ：智能问答核心接口
3. `POST /doc/clear` ：清空 Milvus 整个知识库（运维接口）

## 五、非功能约束
1. 性能：普通请求响应 ≤1.5s，缓存请求 ≤200ms
2. 安全：API Key 配置化存放，禁止硬编码；增加非法字符过滤
3. 规范：代码添加详细注释，全局统一异常处理，返回标准 JSON 格式
